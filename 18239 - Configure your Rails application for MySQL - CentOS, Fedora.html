Setting up Rails application for MySQL server on CentOS and Fedora, MySQL, Ruby, Rails

Configure your Rails application for MySQL - CentOS, Fedora

Difficulty: 1

Time: 30 minutes

This article will assist you to install MySQL™ and MySQL adaptor, create a Rails® Application, configure a database connection, and finally create an application database on a CentOS or Fedora™ distribution.

CentOS or Fedora uses MariaDB as its database system instead of MySQL and MariaDB is based on MYSQL. So here, the term MariaDB will be used in place of MySQL at some instances.

Prerequisites

  • Install Ruby on Rails using RVM - CentOS

Install MariaDB

  1. To install MariaDB and all its dependencies, type:
    sudo yum install mariadb-server mariadb-devel
  2. Start MariaDB by typing the following command:
    sudo systemctl start mariadb
  3. Enable MariaDB to start on boot.
    sudo systemctl enable mariadb.service
  4. Run the MySQL installation process, by typing:
    sudo mysql_secure_installation
  5. After you have finished the above command, a welcome screen displays the following: 
    	NOTE: RUNNING ALL PARTS OF THIS SCRIPT IS RECOMMENDED FOR ALL MySQL
    	      SERVERS IN PRODUCTION USE!  PLEASE READ EACH STEP CAREFULLY!

	In order to log into MySQL to secure it, we'll need the current
	password for the root user.  If you've just installed MySQL, and
	you haven't set the root password yet, the password will be blank,
	so you should just press enter here.
	
	Enter current password for root (enter for none):
  6. Answer the following questions which are prompted by MySQL installer with y: 
    	  Set root password?                     --> y
	  Remove anonymous users?                --> y
	  Disallow root login remotely?          --> y
	  Remove test database and access to it? --> y
	  Reload privilege tables now?           --> y
  7. If you could not set password, use the following command to set the password:
    /usr/bin/mysqladmin -u root password 'new-password'

Install MySQL adapter

  1. To connect Rails application with MySQL database install the MySQL adapter gem using the following command:
    gem install mysql2
  2. Make sure that your gem package manager is up-to-date.
    gem update –system

Set up a Rails application

  1. Make sure that Rails is installed on the machine otherwise verify the version of Rails using:
    rails -v
    #Rails 4.2.4
  2. Create a Rails application, and the use the -d mysql option to set MySQL/MariaDB as the database:
    rails new Your_application -d mysql
  3. Change the directory to the application directory.
    cd Your_application

Set up the database connection

  1. Open the configuration file database.yml to update database credential.
    vim config/database.yml
  2. Set the database credentials to connect with MySQL/MariaDB. The following is an example for database connection. 
    default: &default
  adapter: mysql2
  encoding: utf8
  pool: 5
  username: root
  password: your_password
  socket: /var/lib/mysql/mysql.sock
  3. Save and exit.
    :wq!

Create a new MySQL database

  1. Login to the MySQL console using the following command:
    mysql -uroot -p
  2. Create a new database using the following query:
    create database rails_myapp;

    In MySQL, commands are known as Query Set .

  3. Revoke the privileges.
    FLUSH PRIVILEGES;
  4. Exit from MySQL console by using the following command.
    exit

Next step

By using all the above steps, you’ll be able to configure MariaDB/MySQL Server on CentOS or Fedora for your Rails application.
