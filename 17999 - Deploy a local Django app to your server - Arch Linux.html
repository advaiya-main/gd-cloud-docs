Describes how to setup a local Django app on Arch Linux

Build a LAMP stack for Python - Ubuntu, Debian Linux

Difficulty: 1
Time: 1.50

Django is a Python web framework that allows for rapid development of primarily database-driven web applications. Virtual Environments, or virtualenv, is a Python package that allows development within isolated Python environments. Installing or updating Python packages within one environment will have no impact on other environments.

Install Virtualenv and Install Virtualenvwrapper

VirtualEnv is a Python tool used to create isolated environments for Python in which you can install packages without interfering with the other virtualenvs nor with the system Python's packages.

Install VirtualEnv and virtualenvwrapper with the command below:

sudo pacman -S python2 python2-pip python2-virtualenv python-virtualenvwrapper

VirtualEnvWrapper requires some initial setup

export WORKON_HOME=$HOME/.virtualenvs
source /usr/bin/virtualenvwrapper.sh

Configure Virtual Environment

  1. Create the virtualenv
    mkvirtualenv -p /usr/bin/python2.7 myenv
  2. Activate the virtualenv
    workon myenv
  3. Install some package inside the virtualenv
    pip install django pyproj

Install PostgreSQL, GDAL, GEOS, PostGIS

  1. Install PostgreSQL, PostGIS
    sudo pacman -Sy postgresql postgresql-libs postgis

    Please select the any default values that the install may ask for

  2. Before you install Django, be sure to activate the new virtual environment if not already activated
    workon myapp
  3. Now Django can be installed
    pip install django

You are ready to create a Django project.

Create Django Project

  1. Use the command below to create a django project:
    django-admin.py startproject myproject
  2. This will create a ./myproject directory containing several projects directories and files. Navigate into the project directory.
    cd myproject

    The ./myproject/settings.py file can be modified and a preferred database backend configured. Other settings can be modified as necessary. If no database backend is specified, then Django will use the default sqlite3 backend. You can populate with database with some default Django tables.

You can populate with database with some default Django tables.

python manage.py migrate

The project can now be verified by launching the Django web service.

python manage.py runserver

The service should now be accessible athttp://localhost:8000/. Open another terminal while the "runserver" command is running and type this curl command:

curl http://localhost:8000/

Note: The built-in Django web server is NOT meant to be used in a production environment. It is a tool for development only. Once the application is ready to be deployed into production, consider using a web server such as Apache or Nginx with uWSGI.

Step Five - Configure Django App

Once the project is created and operation, it is time to create an application. An application can be created in the same directory as manage.py.

python manage.py startapp myapp

This will create a ./myapp directory with base files ready for your code.

Next steps

Congratulations! You've now got your app deployed and are ready to build dynamic webpages.
