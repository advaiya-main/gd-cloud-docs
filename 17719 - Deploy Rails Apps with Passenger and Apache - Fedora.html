Deploy Rails Apps with Passenger and Apache - Fedora

Difficulty: 2

Time: 30 minutes

This article guides you on how to deploy Rails Apps with Passenger and Apache in Fedora. Phusion Passenger is one of the fastest app server which handles traffic smoothly than other server with low maintenance You will need at least 1 GB of RAM in a virtual server to run the Passenger commands that consume a lot of memory. It is assumed that the user has sudo privileges to access the virtual server.

This guide includes development library installation and ad hoc compilation on the machine, hence it is not recommended to use this tutorial in a production environment.

Prerequisites

  1. 17322 - Initial Server Setup - Fedora.
  2. 17535 - Install Ruby on Rails using RVM - Fedora
  3. 17714 - Deploy Rails Apps with Passenger and NGINX - Fedora
  4. Fully Qualified Domain Name - FQDN (Optional)

Install Apache

  • Update the system and install Apache.
    sudo yum -y update
    sudo yum -y install curl curl-devel httpd-devel httpd mod_ssl

Install Rails and Passenger

  1. Install Rails and Passenger.
    sudo yum install gem
    gem install rails passenger
  2. Install Passenger package for Apache.
    rvmsudo passenger-install-apache2-module

    This will take some time, so you will need to wait.

  3. Open a new passenger.conf file.
    sudo vim /etc/apache2/conf.d/passenger.conf
  4. Add following lines at end of the files:
    LoadModule passenger_module /home/newuser/.rvm/gems/ruby-2.0.0-p247/gems/passenger-4.0.10/buildout/apache2/mod_passenger.so
    PassengerRoot /home/newuser/.rvm/gems/ruby-2.0.0-p247/gems/passenger-4.0.10
    PassengerDefaultRuby /home/newuser/.rvm/wrappers/ruby-2.0.0-p247/ruby
  5. Save and exit.
    :wq!
  6. Restart Apache service.
    sudo service httpd restart

Set up a Rails application

  1. Install the sqlite development package to store data in database.
    sudo yum install sqlite-devel
  2. Move the test app to Apache's web root folder
    cd /var/www/html
  3. Create a Rails application and go to application directory.
    rails new your-application
    cd your-application
  4. By default, CentOS doesn't have a JavaScript execution environment installed. So to manually install it, type. Open Gemfile in your favourite editor.
    sudo vim Gemfile
  5. Add the following line at end of the file therubyracergem.
    gem 'therubyracer'
  6. Save and exit.
    wq!
  7. Install all the dependencies of Gemfile using the following command:
    bundle install
  8. Migrate the development database for application using the following command:
    rake db:migrate

Connect the Apache Server to the Rails App

  1. Open the configuration file of Apache.
    sudo vim /etc/httpd/conf/http.conf
  2. In this file uncomment the following line
    NameVirtualHost *:80
  3. At the end of this file, add virtual host 
    <VirtualHost *:80>
       ServerName www.mydomain.com //your server name
	 DocumentRoot /var/www/html/HelloWorld/public //your rail application path
   <Directory /var/www/html/HelloWorld/public>
      AllowOverride all
      Options -MultiViews
   </Directory>
</VirtualHost>
  4. Save and exit.
    wq!
  5. Restart the Apache server.
    sudo service httpd restart

Conclusion

In this article, we reviewed the steps involved in setting up your Rails application with Passenger and Apache to make it more secure and reliable in Fedora distribution.
