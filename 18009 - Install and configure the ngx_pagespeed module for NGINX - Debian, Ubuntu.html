Installation and configuration of NGINX with ngx_pagespeed

Install and configure the ngx_pagespeed module for NGINX - Debian, Ubuntu

Difficulty: 1
Time: 15 minutes

NGINX is fast. Pagespeed nakes it faster by optimizing your website. This article descibes how to install ngx_pagespeed, combine it with the NGINX source, and compile and configure the resulting customized NGINX webserver.

Preparation

  1. Update your server.
    sudo apt-get update
  2. Install a C compiler and some other tools
    sudo apt-get install build-essential zlib1g-dev libpcre3 libpcre3-dev unzip

Install ngx_pagespeed

  1. Find the latest NPS version (1.9.32.6) and use it to get the zip file from the archive
    sudo wget https://github.com/pagespeed/ngx_pagespeed/archive/release-1.9.32.6-beta.zip
  2. Unzip the file
    unzip release-1.9.32.6-beta.zip
  3. Navigate to the ngx_pagespeed folder.
    cd ngx_pagespeed-release-1.9.32.6-beta/
  4. Download from Google.
    sudo wget https://dl.google.com/dl/page-speed/psol/1.9.32.6.tar.gz
  5. extract to psol
    tar -xzvf 1.9.32.6.tar.gz

Download NGINX source

  1. Check http://nginx.org/en/download.html for the latest version (1.8.0).
  2. Navigate to your home directory
    cd ~
  3. Download NGINX source
    sudo wget http://nginx.org/download/nginx-1.8.0.tar.gz
  4. Unzip it
    sudo tar -xvzf nginx-1.8.0.tar.gz

Build NGINX from source

  1. Navigate to Nginx source directory.
    cd nginx-1.8.0/
  2. Add the pagespeed module to the NGINX configuration
    sudo ./configure --add-module=$HOME/ngx_pagespeed-release-1.9.32.6-beta
  3. Create a make file
    sudo make
  4. Install from your make file
    sudo make install

Configure NGINX

  1. Open the configuration file for editing
    sudo vim /usr/local/nginx/conf/nginx.conf
  2. Add the following lines to your http context: 
    # http context 
http {
...    
    # pagespeed
    pagespeed on;
    pagespeed DisableRewriteOnNoTransform off;
    pagespeed XHeaderValue "Powered By ngx_pagespeed";
    pagespeed FileCachePath "/var/cache/ngx_pagespeed/";
    pagespeed EnableFilters combine_css,combine_javascript;
...
  3. Modify the Server and location contexts by commenting out the root and index directives in the location contexts and implement them in the server context instead. Changed sections are underlined 
    # http context 
   server {
        listen       80;
        server_name  localhost;
        root   /usr/local/nginx/html;
        index  index.html index.htm;

        #charset koi8-r;

        #access_log  logs/host.access.log  main;

        location / {
        # root   html;    
        # index  index.html index.htm;    
        }

        #error_page  404              /404.html;

        # redirect server error pages to the static page /50x.html
        #
        error_page   500 502 503 504  /50x.html;
        location = /50x.html {
        # root   html;     
        }
 ...
  4. Verify the install by checking the nginx version information
    sudo /usr/local/nginx/sbin/nginx -V
  5. The response will look something like this 
    nginx version: nginx/1.8.0
built by gcc 4.8.4 (Ubuntu 4.8.4-2ubuntu1~14.04)
configure arguments: --add-module=/home/randomcode/ngx_pagespeed-release-1.9.32.6-beta
  6. Start the Nginx service.
    sudo /usr/local/nginx/sbin/nginx

Conclusion

Nginx is now running as a web server enhanced with pagespeed. You can verify by using your browser to navigate to the IP address of your cloudserver, http://IP Address of your cloudserver .
