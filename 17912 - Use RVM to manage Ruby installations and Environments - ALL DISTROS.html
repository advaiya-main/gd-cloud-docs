Managing Ruby installations and environments using RVM, Ruby, Rails

Use RVM to manage Ruby installations and environments - ALL DISTROS

Difficulty: 2
Time: 3 hours

Ruby® is a very popular object-oriented, general-purpose programming language that has Ruby on Rails™ or simply Rails, which is written in Ruby. Rails are an open source Ruby framework for developing database-backed web applications.

Generally there is a need for different versions of Ruby installed in the same server for different projects. We can achieve this with the help of Ruby Version Manager (RVM).

In this article, we will be discussing on managing Ruby installations using RVM. We will continue assuming that you have already installed RVM on server.

RVM Command

After RVM is installed in server, you will have rvm command to manage the same.

Syntax:

rvm command_options command ruby_to_act_on
Eg: rvm install ruby

Commands and their purpose

TASK COMMAND
To get the list of different command options available rvm help
To get details about a particular command rvm help command
To find the installed packages in RVM rvm info
To update RVM rvm get stable
To enable Tab completion in .bashrc file [[ -r $rvm_path/scripts/completion ]] && . $rvm_path/scripts/completion

Install and uninstall Rubies using RVM

TASK COMMAND
To get the list of available Rubies to install rvm list known
To install a selected Ruby version rvm install ruby_version
To get the list of installed Rubies rvm list
To switch between the multiple versions of Ruby installed rvm use other_ruby_version
To set a particular Ruby version as default rvm command ruby_version --default
To switch to a default Ruby rvm default
To uninstall a particular Ruby version rvm uninstall ruby_version

Managing Gemsets with RVM

Gems are packages which contains package information along with files to install. The gem command is used to build and install Gem packages. Gemset is a set of gems collected together.

When you install Ruby using RVM, it comes with two gemsets:

  1. Default : This gemset is used when no other gemset is specified
  2. Global : This gemset is inherited by every other gemset that is used

Commands for managing Gemsets

TASK COMMAND
To see the available gemsets for current Ruby rvm gemset list
To see gemsets for more than one Ruby installations rvm gemset list_all
To create a gemset rvm gemset create gem_test
To copy a current gemset to a new gemset rvm gemset copy default gem_test
To use a particular gemset rvm gemset use gem_test
To set default version for Ruby, when you have more than one gemset installed rvm --default use ruby_version@gem_test
To delete a gemset rvm gemset delete gem_test
To create project specific configurations to specify which Ruby version and gemset to use in a .rvmrc file rvm ruby_version@gemset

Configurations

  • RVM has its default configuration file in:
    $ ~/.rvm/config/db

    If RVM is upgraded, then the customizations made to this file will be lost forever. Therefore custom configurations can be made in '~/.rvm/user/db' so that the settings will retain.
