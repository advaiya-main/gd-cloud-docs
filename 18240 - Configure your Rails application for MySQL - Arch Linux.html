Setting up Rails application for MySQL server on Arch Linux, MySQL, Ruby, Rails

Configure your Rails application for MySQL - Arch Linux

Difficulty: 1

Time: 30 minutes

This article will explain how to install MySQL™ and MySQL adaptor, create a Rails® Application, configure a database connection, and finally, how to create an application database on Arch Linux™.

Arch Linux uses MariaDB as its database system instead of MySQL and MariaDB is based on MYSQL. So here, the term MariaDB will be used in place of MySQL at some instances.

Prerequisite

  • 17537 - Install Ruby on Rails using RVM - Arch Linux

Install MySQL

  1. Download and install the MariaDB database server.
    sudo pacman -Sy
    sudo pacman -S mariadb
  2. Initialize MySQL database directories.
    sudo mysql_install_db --user=mysql --basedir=/usr --datadir=/var/lib/mysql
  3. Start the MySQL using the following command:
    sudo systemctl start mysqld
  4. Enable MySQL for boot start-up using the following command:
    sudo systemctl enable mysqld
  5. You need to tell MySQL to create a database directory structure where it will store its information. You can do this by typing:
    sudo mysql_install_db
  6. Run the MySQL installation process, by typing:
    sudo mysql_secure_installation
  7. After you have finished the above command, a welcome screen displays the following: 
    NOTE: RUNNING ALL PARTS OF THIS SCRIPT IS RECOMMENDED FOR ALL MySQL
      SERVERS IN PRODUCTION USE!  PLEASE READ EACH STEP CAREFULLY!
      In order to log into MySQL to secure it, we'll need the current
      password for the root user.  If you've just installed MySQL, and
      you haven't set the root password yet, the password will be blank,
      so you should just press enter here.

     Enter current password for root (enter for none):
  8. Answer the following questions which are prompted by MySQL installer with y: 
    	  Set root password?                     --> y
	  Remove anonymous users?                --> y
	  Disallow root login remotely?          --> y
	  Remove test database and access to it? --> y
	  Reload privilege tables now?           --> y
  9. If you could not set password, use the following command to set the password:
    /usr/bin/mysqladmin -u root password 'new-password'
  10. To enable remote connections, open the MySQL database configuration my.cnf and edit it using the vim text editor.
    sudo vim /etc/mysql/my.cnf

    Add the following line at the end of the file:

    [mysqld]
    ..
    .
    bind-address = 0.0.0.0
  11. Restart the MySQL Sever to effect the change.
    sudo systemctl start mysqld

Set up a Rails application

  1. Make sure that Rails is installed on machine otherwise install, to verify use the following command::
    rails -v
    #Rails 4.2.4
  2. Create a Rails application, and the use the -d mysql option to set MySQL/MariaDB as the database.
    rails new your-application -d mysql
  3. Change the directory to the application directory.
    cd your-application

Set up the database connection

  1. Open your application database configuration file.
    vi config/database.yml
  2. Set the database credentials to connect with MySQL. The following is an example for database connection. 
    default: &default
  adapter: mysql2
  encoding: utf8
  pool: 5
  username: root
  password: your_password
  socket: /var/lib/mysql/mysql.sock
  3. Save and exit.
    :wq!
